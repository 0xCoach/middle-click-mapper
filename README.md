# 鼠标中键映射助手

**中文 | [English](README_EN.md)**

一个轻量级的 macOS 后台工具，把**鼠标中键**映射成**任意自定义快捷键**。

按住中键即触发对应的快捷键，松开即结束——非常适合把中键变成语音输入、全局搜索、截图等需要「按住式触发」的功能。

## 工作原理

App 通过 `CGEventTap` 监听全局鼠标事件，捕获到中键（鼠标 button 2）按下/松开后，用 `CGEvent` 模拟一次配置好的键盘快捷键。

```
鼠标中键按下 ──▶ 模拟按下「快捷键」
鼠标中键松开 ──▶ 模拟松开「快捷键」
```

默认映射为「右 Command」，搭配语音输入类 App（许多会把语音输入快捷键设为右 Command）可开箱即用。你也可以在界面里把映射改成任意组合键。

## 默认配置

| 项 | 默认值 |
|----|--------|
| 监听的鼠标按键 | 中键（button 2） |
| 映射的快捷键 | 右 Command |
| 触发方式 | 按住触发，松开结束 |

## 安装（给使用者）

本 App 使用 **ad-hoc 签名**（无付费开发者证书），首次打开需要绕过一次 macOS Gatekeeper，之后即可正常使用。

1. 打开 `鼠标中键映射助手.dmg`，把 `鼠标中键映射助手.app` 拖到 `Applications`。
2. **首次打开**：在 App 图标上 **右键 → 打开**（或触控板双指轻点）。
   > ⚠️ 不能直接双击。直接双击会提示「无法打开 / 已损坏」——这是 macOS 对未签名 App 的正常拦截，不是真的损坏。右键打开后，弹出的对话框里会出现「打开」按钮，点它即可。**这次之后系统会记住，以后双击就能直接开。**
3. 打开 App，点击窗口里的 **「前往授权」** 按钮（或去 `系统设置 → 隐私与安全性 → 辅助功能`）。
4. 开启「鼠标中键映射助手」的辅助功能开关。
5. 回到 App 窗口，确认状态卡片显示绿点（已授权 · 监听运行中），按一下鼠标中键测试。

## 使用

- **打开界面**：点击菜单栏的 **⌘ 图标**，或点击 Dock 图标。
- **改映射快捷键**：点「设置快捷键」，按下你想要的组合键（单独的修饰键如右 Command 也可以）。
- **开机自启**：在窗口里勾选「开机自动启动」，登录项里会显示为「鼠标中键映射助手」。
- **搭配语音输入类 App**：把该 App 的语音输入快捷键设为「右 Command」，中键即变成「按住说话」。

## 构建（给开发者）

```sh
./build.sh
```

构建脚本会：

1. 用 `swiftc` 编译 `src/MiddleClickMapper.swift`，生成 `dist/鼠标中键映射助手.app`。
2. 用 **ad-hoc 签名**（`codesign -s -`）+ Hardened Runtime 签名。
3. 清理本机失效的 TCC 授权记录（见下文说明）。
4. 打包成 `dist/鼠标中键映射助手.dmg`（内含 App、Applications 快捷方式、背景图）。

## 关于签名的几点说明

**为什么用 ad-hoc 签名？**
本 App 面向免费分发，没有 Apple 付费开发者账号。ad-hoc 签名的代价是接收方首次打开需「右键 → 打开」一次。若想做到双击直接开（无需任何手动操作），需要 Apple 开发者账号做 Developer ID 签名 + 公证（notarization）。

**为什么每次重新构建后要重新授权？**
ad-hoc 签名的二进制 cdhash 每次构建都会变，而 macOS 的辅助功能权限（TCC）是按「二进制指纹」记录的——指纹一变，旧授权就失效。`build.sh` 已自动清理失效记录（`tccutil reset`），构建后重启 App 并重新授权一次即可。

> 注意：这一摩擦只影响**本机反复构建**的开发场景。分发给别人的 DMG 是单一固定版本，接收方机器是干净的，只需授权一次，不会出现这个问题。

## 技术要点

- **`CGEventTap`**（`.cgSessionEventTap`）全局监听中键，过滤 `otherMouseDown/Up`（button 2）。
- **`CGEvent`** 模拟按键，精确处理修饰键的按下顺序与 flags，避免组合键错乱。
- **`LSUIElement = true`** 让 App 作为后台 agent 运行（默认不占 Dock），但通过动态切换 activation policy 兼顾「Dock 可点开」+「菜单栏常驻」双入口。
- **LaunchAgent 开机自启**直接执行 App 二进制（而非 `/usr/bin/open` 中转），确保登录项显示真实名字。
